## Overview

YouTube와 웹페이지의 추출 결과 UI가 서로 다른 구조(썸네일/채널 유무, 간격 차이)로 표시되는 문제를 해결한다. 두 타입 모두 동일한 카드 구조(썸네일 + 제목 + 출처 + 복사 버튼 + 프롬프트 + 본문)로 통일한다.

## 시나리오

### 1. 웹페이지 결과에 OG 이미지 표시

[상황] 웹페이지 URL을 추출하여 결과가 표시된 상태에서
[동작] OG 이미지가 있으면 제목 위에 썸네일로 표시된다

성공 기준:
- [ ] OG 이미지가 있는 웹페이지 추출 -> 제목 위에 이미지가 16:9 비율로 표시된다
- [ ] OG 이미지가 없는 웹페이지 추출 -> 썸네일 영역이 표시되지 않는다

### 2. 웹페이지 결과에 출처 표시

[상황] 웹페이지 URL을 추출하여 결과가 표시된 상태에서
[동작] 제목 아래에 출처(author 또는 도메인명)가 표시된다

성공 기준:
- [ ] author가 있는 웹페이지 추출 -> 제목 아래에 author 텍스트가 표시된다
- [ ] author가 없는 웹페이지 추출 -> 제목 아래에 도메인명(예: "example.com")이 표시된다

### 3. YouTube와 웹페이지 결과의 레이아웃 통일

[상황] YouTube 또는 웹페이지 URL을 추출하여 결과가 표시된 상태에서
[동작] 두 타입 모두 동일한 요소 순서(썸네일 -> 제목 -> 출처 -> 복사 버튼 -> 프롬프트 -> 본문)와 동일한 간격으로 표시된다

성공 기준:
- [ ] YouTube 결과 -> 썸네일, 제목, 채널명, 복사 버튼, 프롬프트, 본문 순서로 표시된다
- [ ] 웹페이지 결과(OG 이미지 있음) -> 썸네일, 제목, 출처, 복사 버튼, 프롬프트, 본문 순서로 표시된다
- [ ] 웹페이지 결과(OG 이미지 없음) -> 제목, 출처, 복사 버튼, 프롬프트, 본문 순서로 표시된다
- [ ] YouTube와 웹페이지의 전체 간격(gap)이 동일하다

### 4. 백엔드에서 웹페이지 메타 정보 반환

[상황] 웹페이지 URL로 콘텐츠 추출 API를 호출할 때
[동작] 응답에 image(OG 이미지)와 author 필드가 포함된다. author가 없으면 URL에서 추출한 도메인명이 source로 반환된다

성공 기준:
- [ ] OG 이미지가 있는 웹페이지 -> 응답에 thumbnail 필드가 포함된다
- [ ] author 메타가 있는 웹페이지 -> 응답에 author 값이 포함된다
- [ ] author 메타가 없는 웹페이지 -> 응답에 source 필드로 도메인명이 반환된다

## 범위

### 포함

- 웹페이지 추출 시 image, author 필드 반환 (extract.ts 수정)
- author 없을 때 도메인명을 source로 반환
- 프론트엔드 결과 UI를 type 분기 없이 통일된 구조로 변경
- gap 값 통일

### 제외

- 썸네일 이미지 크기/비율 변경 — 기존 16:9 유지
- 새로운 UI 컴포넌트 추가 — 기존 컴포넌트 활용
- 웹페이지 favicon 표시 — MVP 범위 외

## 전제 조건

- Defuddle 라이브러리가 웹페이지에서도 image, author를 파싱할 수 있다
