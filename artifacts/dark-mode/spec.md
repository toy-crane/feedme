## Overview
사용자가 다크 모드와 라이트 모드를 전환할 수 있는 토글 버튼을 우측 상단에 고정 배치한다. 최초 방문 시 OS 테마를 따르고, 사용자가 수동 전환하면 선택이 유지된다.

## 시나리오

### 1. 다크 모드 토글 표시
[상황] 사용자가 feedme 페이지에 접속하면
[동작] 우측 상단에 테마 전환 토글이 fixed로 표시된다

성공 기준:
- [ ] 페이지 접속 시 -> 우측 상단에 토글 버튼이 항상 표시된다
- [ ] 페이지를 스크롤해도 -> 토글이 화면에 고정되어 있다

### 2. 라이트에서 다크 모드 전환
[상황] 라이트 모드 상태에서 토글을 클릭하면
[동작] 다크 모드로 전환되고 html에 dark 클래스가 적용된다

성공 기준:
- [ ] 라이트 모드에서 토글 클릭 -> 배경이 어두운 색으로 변경된다
- [ ] 라이트 모드에서 토글 클릭 -> html 요소에 class="dark"가 적용된다

### 3. 다크에서 라이트 모드 전환
[상황] 다크 모드 상태에서 토글을 클릭하면
[동작] 라이트 모드로 전환되고 html에서 dark 클래스가 제거된다

성공 기준:
- [ ] 다크 모드에서 토글 클릭 -> 배경이 밝은 색으로 변경된다

### 4. 시스템 테마 자동 반영
[상황] 사용자가 처음 방문하면 (localStorage에 테마 설정 없음)
[동작] OS의 다크 모드 설정을 따른다

성공 기준:
- [ ] OS 다크 모드 설정 + 첫 방문 -> 다크 모드로 표시된다
- [ ] OS 라이트 모드 설정 + 첫 방문 -> 라이트 모드로 표시된다

### 5. 사용자 선택 유지
[상황] 사용자가 토글로 테마를 변경한 후 페이지를 새로고침하면
[동작] 마지막으로 선택한 테마가 유지된다

성공 기준:
- [ ] 다크 모드로 전환 후 새로고침 -> 다크 모드가 유지된다

### 6. 다크 모드 코드 하이라이팅
[상황] 다크 모드에서 코드 블록이 포함된 콘텐츠를 확인하면
[동작] 다크 테마에 맞는 코드 하이라이팅이 적용된다

성공 기준:
- [ ] 다크 모드에서 코드 블록 확인 -> 어두운 배경의 코드 하이라이팅이 표시된다

## 범위

### 포함
- 우측 상단 fixed 토글 버튼 (Sun/Moon 아이콘)
- light/dark 모드 전환
- OS 테마 자동 감지 (enableSystem)
- 사용자 선택 localStorage 저장 (next-themes 기본 동작)
- 다크 모드 코드 하이라이팅 테마

### 제외
- 3단 모드(light/dark/system) 선택 UI — 토글로 2단 전환만 지원, 복잡도 절감
- 애니메이션 전환 효과 — MVP 범위 외
- 커스텀 테마 색상 — 기존 CSS 변수의 .dark 클래스 활용

## 전제 조건
- next-themes 패키지 설치 완료 (이미 설치됨)
- globals.css에 .dark CSS 변수 정의 완료 (이미 정의됨)
- ThemeProvider가 layout.tsx에 설정 완료 (이미 설정됨, enableSystem 변경 필요)
