# Input Form 로딩 버튼 및 에러 상태 개선

## Overview

URL 입력 폼의 로딩/에러 UX를 개선한다. 빈 입력 시 버튼을 비활성화하여 불필요한 에러 메시지를 없애고, 로딩 중에는 버튼에 스피너를 표시하며, 서버 에러는 입력 필드 하단 인라인 텍스트로 표시한다.

## 시나리오

### 1. 빈 입력 시 버튼 비활성화

[상황] URL 입력란이 비어 있는 초기 화면에서
[동작] '가져오기' 버튼이 disabled 상태로 표시된다

성공 기준:
- [ ] URL 입력란이 비어 있으면 '가져오기' 버튼이 disabled 상태
- [ ] 유효한 URL을 입력하면 버튼이 활성화됨

### 2. 잘못된 URL 입력 시 버튼 비활성화

[상황] URL 입력란에 유효하지 않은 텍스트가 입력된 상태에서
[동작] '가져오기' 버튼이 disabled 상태로 표시된다

성공 기준:
- [ ] "not-a-url" 입력 시 '가져오기' 버튼이 disabled 상태
- [ ] "https://example.com"으로 수정하면 버튼이 활성화됨

### 3. 로딩 중 버튼 스피너 표시

[상황] 유효한 URL을 입력하고 '가져오기' 버튼을 클릭한 직후
[동작] 버튼에 스피너 아이콘만 표시되고 버튼은 disabled된다

성공 기준:
- [ ] 로딩 중 버튼 텍스트가 사라지고 스피너 아이콘이 표시됨
- [ ] 버튼이 disabled 상태
- [ ] 기존 별도 로딩 영역("불러오는 중...")은 제거됨

### 4. 서버 에러 인라인 표시

[상황] 접근 불가능한 URL을 입력하고 '가져오기' 버튼을 클릭한 후 서버 에러가 반환된 상태에서
[동작] 입력 필드 하단에 빨간 인라인 텍스트로 에러 메시지가 표시된다

성공 기준:
- [ ] 접근 불가 URL → 입력 필드 하단에 "페이지에 접근할 수 없습니다" 빨간 텍스트 표시
- [ ] 자막 없는 YouTube URL → "자막을 찾을 수 없습니다" 빨간 텍스트 표시
- [ ] 네트워크 오류 → "네트워크 오류가 발생했습니다" 빨간 텍스트 표시
- [ ] 기존 Alert 컴포넌트는 제거됨

### 5. URL 입력 시 에러 자동 해제

[상황] 에러 메시지가 표시된 상태에서
[동작] 사용자가 URL 입력란을 수정하면 에러 메시지가 사라진다

성공 기준:
- [ ] 에러 표시 중 입력란에 타이핑하면 에러 텍스트가 즉시 사라짐

## 범위

### 포함

- 버튼 disabled 조건 변경 (빈 입력 + 잘못된 URL + 로딩 중)
- 로딩 중 버튼 스피너 아이콘
- 에러 표시를 Alert에서 인라인 텍스트로 변경
- URL 입력 시 에러 자동 해제

### 제외

- 에러 메시지 문구 변경 — 기존 서버 에러 메시지 그대로 유지
- 입력 필드 테두리 색상 변경 — 변경 비용 대비 효과 낮음
- 클라이언트 유효성 에러 메시지 — 버튼 비활성화로 대체

## 전제 조건

- 기존 `isValidUrl()` 유틸리티 활용
- 기존 InputGroup 컴포넌트 구조 유지
