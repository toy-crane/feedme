## Overview

마크다운 미리보기의 렌더링 품질과 레이아웃을 개선한다. 현재 카드(테두리+높이 제한) 안에 갇혀서 콘텐츠를 한눈에 보기 어렵고, 코드 블록 등 마크다운 요소가 제대로 렌더링되지 않는 문제를 해결한다.

## 시나리오

### 1. 높이 제한 없이 전체 콘텐츠 표시

[상황] 웹페이지 URL을 추출하여 마크다운 미리보기가 표시된 상태에서
[동작] 긴 콘텐츠가 추출되면 블로그처럼 전체 콘텐츠가 높이 제한 없이 페이지에 펼쳐진다

성공 기준:
- [ ] 웹페이지 추출 → 미리보기 영역에 max-height 스크롤이 없고 전체 콘텐츠가 표시된다
- [ ] YouTube 자막 추출 → 자막 영역에 max-height 스크롤이 없고 전체 자막이 표시된다

### 2. 테두리 없는 자연스러운 레이아웃

[상황] 콘텐츠가 추출되어 미리보기가 표시된 상태에서
[동작] 미리보기 영역에 테두리(border)가 없이 블로그처럼 자연스럽게 콘텐츠가 표시된다

성공 기준:
- [ ] 웹페이지 미리보기 영역에 border 스타일이 적용되지 않는다
- [ ] YouTube 자막 영역에 border 스타일이 적용되지 않는다

### 3. 콘텐츠 영역 너비 확대

[상황] feedme 페이지에 접속한 상태에서
[동작] 콘텐츠 영역의 최대 너비가 기존(max-w-lg, ~512px)보다 넓어져(max-w-2xl, ~672px) 코드 블록이나 테이블을 더 잘 표시한다

성공 기준:
- [ ] 콘텐츠 영역의 최대 너비가 max-w-2xl(672px) 이상이다

### 4. GFM(GitHub Flavored Markdown) 전체 지원

[상황] 취소선, 테이블, 태스크리스트, 자동 링크가 포함된 마크다운이 추출된 상태에서
[동작] 모든 GFM 요소가 올바르게 렌더링된다

성공 기준:
- [ ] `~~취소선~~` → 취소선이 적용된 텍스트로 렌더링된다 (`<del>` 태그)
- [ ] 마크다운 테이블 → HTML `<table>`로 렌더링된다
- [ ] `- [x] 완료` → 체크된 체크박스로 렌더링된다
- [ ] `https://example.com` → 클릭 가능한 링크로 렌더링된다

### 5. 코드 블록 구문 하이라이팅

[상황] 언어가 지정된 코드 블록(```javascript 등)이 포함된 마크다운이 추출된 상태에서
[동작] 코드 블록에 구문 하이라이팅이 적용되어 키워드, 문자열 등이 색상으로 구분된다

성공 기준:
- [ ] ` ```javascript\nconst x = 1;``` ` → 코드 블록 내 키워드가 색상으로 구분되어 표시된다
- [ ] 언어 미지정 코드 블록 → 하이라이팅 없이 기본 코드 블록 스타일로 표시된다

## 범위

### 포함

- 미리보기 높이 제한(max-h) 제거
- 미리보기 테두리(border) 제거
- 콘텐츠 영역 너비 확대 (max-w-lg → max-w-2xl)
- remark-gfm 플러그인 추가 (취소선, 테이블, 태스크리스트, 자동 링크)
- rehype-highlight 플러그인 추가 (코드 블록 구문 하이라이팅)

### 제외

- 마크다운/원문 토글 기능 — 현재 스펙의 '편집 없이 바로 복사' 원칙 유지
- 복사 버튼 위치 변경 — 현재 위치(미리보기 상단) 유지
- 다크 모드별 하이라이팅 테마 — MVP 범위 외

- remark-gfm, rehype-highlight 패키지 추가 필요 (새 외부 의존성)
