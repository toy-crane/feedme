Best Practices for Claude Code

복사

Claude Code는 자율적인 코딩 에이전트입니다. 효과적으로 사용하기 위한 모범 사례를 소개합니다.

프로젝트 설정

CLAUDE.md 파일을 프로젝트 루트에 만들어 Claude에게 컨텍스트를 제공하세요.

// CLAUDE.md 예시
const config = {
name: "my-project",
version: "1.0.0",
port: 3000
};

효과적인 프롬프트 작성

구체적이고 명확한 프롬프트를 작성하면 더 좋은 결과를 얻을 수 있습니다.

좋은 프롬프트는 문제의 맥락, 기대 결과, 제약 조건을 포함합니다.

명령어 구조

다음 명령어를 활용할 수 있습니다:

  • /help — 도움말 표시
  • /commit — 변경 사항 커밋
  • /review — 코드 리뷰
$ claude --help $ claude /commit $ claude /review

디버깅 팁

에러가 발생하면 로그를 확인하고, 필요한 경우 --verbose 플래그를 추가하세요.

  1. 에러 메시지 확인
  2. 관련 파일 검토
  3. 테스트 실행
Notes

웹페이지 추출 결과 화면 — 기존 대비 변경점

  • 레이아웃 너비: max-w-lg(512px) → max-w-2xl(672px)
  • 미리보기 영역: border 제거, max-h-96 제거 → 전체 펼침
  • 코드 블록: 구문 하이라이팅 적용 (언어 지정 시)
  • 언어 미지정 코드 블록: 하이라이팅 없이 기본 스타일
  • 인터랙션: 복사 버튼은 상단 고정 유지

채널명

안녕하세요, 오늘은 Claude Code의 모범 사례에 대해 이야기해보겠습니다.

첫 번째로 프로젝트 설정에 대해 알아보겠습니다. CLAUDE.md 파일을 만들어서 프로젝트의 컨텍스트를 제공하는 것이 중요합니다.

두 번째는 효과적인 프롬프트 작성법입니다. 구체적이고 명확한 프롬프트를 작성하면 더 좋은 결과를 얻을 수 있습니다.

세 번째로 디버깅 팁을 소개합니다. 에러가 발생하면 로그를 먼저 확인하시기 바랍니다.

네 번째는 테스트 작성에 대한 가이드입니다. TDD 방식을 활용하면 안정적인 코드를 작성할 수 있습니다.

다섯 번째로 협업 시 유의사항을 설명드리겠습니다. 커밋 메시지를 명확하게 작성하는 것이 중요합니다.

마지막으로, Claude Code를 활용한 코드 리뷰 방법을 소개합니다. /review 명령어를 사용해보세요.

감사합니다. 더 자세한 내용은 공식 문서를 참고해주세요.
